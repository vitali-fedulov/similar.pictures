About

This service finds similar photos on your computer (Windows, Mac OS and Linux). One image allows finding similar images, for example originals, to a source photo. All similarities finds multiple groups of similar images in a folder or disk (example).

Only near duplicates can be found. Specifically: exact or resized pictures, images with small color changes, minor watermarks. It will not find stronly cropped or rotated images, similar objects, or same objects taken from different views, ligting conditions etc.

How to use

  • Use desktop browsers (it is unlikely to work on smarphones).
  • Click a button below and select an image or a directory.
  • Click "Open" or "Upload" (images are NOT UPLOADED anywhere, but read-access is given to all files of selected directory).
  • To get an idea of what is considered "similar", try "All similarities" first. Do not select a folder with too many files (>10000-50000), as it will take very long initial selection time and more memory to process.

Image types supported

JPG, PNG, BMP, WEBP, GIF (static and animated) of maximum size 40MB per image. Transparency is ignored in image comparison.

Open-source algorithm

Read about this photo comparison algorithm and check out image similarity in Go (Github) for your company or project. The open-source Golang code implements the image comparison function. The code does not include the duplicate image finder (clustering algorithm).

Precision

Parameters are set to find reasonably high number of near-similar images with acceptable speed and low false positives. The idea is to offer ready-to-use solution for majority of typical cases without many customizable parameters.

Images are compared by perceptual similarity in terms of color and brightness across the whole image. Partial occlusions, e.g. watermarks, impact perceptual similarity. If the perceptual changes are small, the pictures are considered similar. Rotated and mirrored images are considered distinct.
